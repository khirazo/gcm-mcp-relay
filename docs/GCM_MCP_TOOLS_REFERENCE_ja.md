# GCM MCP Relay - ツールリファレンス

IBM Guardium Cryptography Manager (GCM) MCP Relayが提供する26個のツールの完全なリファレンスです。

## 📊 ツール概要

- **総ツール数**: 26個
- **読み取り専用ツール**: 22個 (安全)
- **状態変更ツール**: 4個 (要注意)

---

## 1️⃣ ポリシー管理 (Policy Management)

セキュリティポリシーの検索、表示、作成を行うツール群。

### 読み取り専用ツール

#### `search_policies`
ポリシーをフィルタリングして検索します。ポリシータイプ、アクティブ状態、作成者などで絞り込み可能。

**用途**: ポリシー一覧の取得、特定条件でのポリシー検索

#### `fetch_policy_by_id`
ポリシーIDを指定して、特定のポリシーの詳細情報を取得します。

**用途**: 個別ポリシーの詳細確認、ポリシー設定の検証

### 状態変更ツール

#### `create_policy` ⚠️ **HIGH RISK**
新しいセキュリティポリシーを作成します。

**用途**: 新規ポリシーの定義と登録  
**アクセス**: admin プロファイルのみ  
**注意**: システム全体のセキュリティ設定に影響

---

## 2️⃣ 違反管理 (Violation Management)

ポリシー違反の照会とチケット管理を行うツール群。

### 読み取り専用ツール

#### `get_violation_by_id`
違反IDを指定して、特定の違反の詳細情報を取得します。

**用途**: 個別違反の詳細確認、違反内容の分析

#### `fetch_policy_violations_ticket`
ポリシー違反に関連するチケットを検索・一覧表示します。

**用途**: 違反チケットの管理、チケットステータスの追跡

#### `policy_violations_dashboard`
違反の統計情報とダッシュボードデータを取得します。

**用途**: 違反の全体像把握、レポート作成

### 状態変更ツール

#### `create_violation_ticket` ⚠️ **MODERATE RISK**
ポリシー違反に対するチケットを作成します。JiraやServiceNowなどの外部システムと連携可能。

**用途**: 違反の追跡、是正措置の管理  
**アクセス**: ops, admin プロファイル  
**注意**: 外部チケットシステムにレコードを作成

---

## 3️⃣ ITアセット管理 (IT Asset Management)

データベース、サーバー、アプリケーションなどのITアセットを管理するツール群。

### 読み取り専用ツール

#### `get_filters_by_it_assets`
ITアセットの検索に使用可能なフィルタ条件を取得します。

**用途**: 検索条件の確認、動的フィルタの構築

#### `fetch_detailed_asset_list_by_it_assets`
ITアセットの詳細リストをフィルタリングして取得します。ページネーション対応。

**用途**: アセット一覧の取得、インベントリ管理

#### `fetch_individual_asset_detail_by_it_assets`
特定のITアセットの詳細情報を取得します。複数アセットの一括取得も可能。

**用途**: 個別アセットの詳細確認、アセット属性の検証

#### `get_category_metadata_by_it_assets`
ITアセットカテゴリのメタデータ情報を取得します。

**用途**: アセット属性の理解、スキーマ情報の取得

---

## 4️⃣ 暗号オブジェクト管理 (Crypto Object Management)

証明書、鍵、プロトコルなどの暗号オブジェクトを管理するツール群。

### 読み取り専用ツール

#### `get_filters_by_crypto_objects`
暗号オブジェクトの検索に使用可能なフィルタ条件を取得します。

**用途**: 検索条件の確認、暗号資産の分類

#### `fetch_detailed_asset_list_by_crypto_objects`
暗号オブジェクトの詳細リストをフィルタリングして取得します。証明書、鍵、プロトコルを含む。

**用途**: 暗号資産の一覧取得、暗号インベントリ管理

#### `fetch_individual_asset_detail_by_crypto_objects`
特定の暗号オブジェクトの詳細情報を取得します。

**用途**: 個別暗号オブジェクトの詳細確認、証明書/鍵の検証

#### `get_category_metadata_by_crypto_objects`
暗号オブジェクトカテゴリのメタデータ情報を取得します。

**用途**: 暗号オブジェクト属性の理解、スキーマ情報の取得

#### `fetch_bulk_vulnerable_crypto_objects`
脆弱性のある暗号オブジェクトを一括で取得します。違反情報とチケット作成可否を含む。

**用途**: 脆弱性スキャン、リスク評価

#### `get_vulnerable_crypto_objects_count`
脆弱な暗号オブジェクトのタイプ別カウントを取得します。

**用途**: 脆弱性の統計、ダッシュボード表示

---

## 5️⃣ アセット管理 (Asset Management)

アセットグループとメタデータを管理するツール群。

### 読み取り専用ツール

#### `get_asset_groups`
利用可能なすべてのアセットグループを取得します。

**用途**: アセットグループの一覧表示、グループ管理

#### `fetch_asset_metadata`
特定のアセットカテゴリのメタデータを取得します。

**用途**: アセット属性の理解、データモデルの確認

---

## 6️⃣ 証明書ライフサイクル管理 (Certificate Lifecycle)

証明書の権限管理と更新を行うツール群。

### 読み取り専用ツール

#### `get_certificate_permissions`
特定の証明書に対する権限情報を取得します。

**用途**: アクセス制御の確認、権限監査

#### `get_vault_details`
Vault統合の詳細情報を取得します。

**用途**: Vault設定の確認、統合状態の検証

#### `get_certificate_details`
証明書IDを指定して、証明書の詳細情報を取得します。

**用途**: 証明書情報の確認、有効期限の監視

### 状態変更ツール

#### `renew_ca_signed_certificate` ⚠️ **HIGH RISK**
CA署名証明書を更新します。自動デプロイオプション付き。

**用途**: 証明書の更新、証明書ライフサイクル管理  
**アクセス**: admin プロファイルのみ  
**注意**: 本番環境の証明書に影響

#### `renew_self_signed_certificate` ⚠️ **HIGH RISK**
自己署名証明書を作成または更新します。

**用途**: 自己署名証明書の管理、テスト環境の証明書作成  
**アクセス**: admin プロファイルのみ  
**注意**: 証明書の作成・更新を実行

---

すべての統合設定をページネーション付きで取得します。

**用途**: 統合一覧の表示、統合タイプの確認

## 8️⃣ ユーザー管理 (User Management)

ユーザー情報を照会するツール群。

### 読み取り専用ツール

#### `get_user_details_by_username`
ユーザー名またはメールアドレスでユーザー詳細を取得します。

**用途**: ユーザー情報の確認、アカウント検証

## 🔒 セキュリティとアクセス制御

### プロファイル別アクセス

| プロファイル | 読み取り専用 | 状態変更(MODERATE) | 状態変更(HIGH) |
|------------|------------|------------------|---------------|
| **readonly** | ✅ 全22個 | ❌ | ❌ |
| **ops** | ✅ 全22個 | ✅ 1個 | ❌ |
| **admin** | ✅ 全22個 | ✅ 1個 | ✅ 3個 |

### リスクレベル

- **SAFE** (22個): データの読み取りのみ、システム状態を変更しない
- **MODERATE** (1個): チケット作成など、限定的な状態変更
- **HIGH** (3個): ポリシー作成、証明書更新など、重要な状態変更

## 📝 使用例

### 基本的な照会
```bash
# ポリシーを検索
/gcm search_policies でアクティブなポリシーを検索

# 証明書の詳細を取得
/gcm get_certificate_details で証明書情報を確認
```

### 違反管理
```bash
# 違反ダッシュボードを表示
/gcm policy_violations_dashboard で違反統計を取得

# 違反チケットを作成 (ops/admin)
/gcm create_violation_ticket で新規チケットを作成
```

### 証明書管理
```bash
# 証明書を更新 (admin)
/gcm renew_ca_signed_certificate で証明書を更新
```

## 🔍 技術詳細

- **認証**: OAuth2/OIDC (Keycloak経由)
- **トランスポート**: stdio (Phase 1) / HTTP (Phase 2)
- **監査**: すべての操作を `/logs/audit.jsonl` に記録
- **デプロイ**: Docker Compose経由
